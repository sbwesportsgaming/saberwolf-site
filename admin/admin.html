Organizações de Torneios

Nesta etapa, o Admin Master libera apenas a porta de entrada: o usuário autorizado poderá criar uma organização real na próxima fase. A permissão não transforma o usuário em Admin do site e não concede acesso a outras organizações.

  • Usuário comum SBW sem permissão: não cria organização.
  • Usuário autorizado pela -SBW-: pode acessar “Criar organização”.
  • Permissão revogada: perde a porta de entrada.
  • Criação/edição real da organização fica para a v1.6.53.

Diagnóstico Supabase/RLS

Verifica se as tabelas sensíveis estão acessíveis para a conta atual e ajuda a identificar onde o Supabase/RLS ainda precisa ser ajustado.

Este diagnóstico não altera dados. Ele faz leituras seguras e registra se a tabela respondeu, se foi bloqueada por RLS ou se ainda não existe no schema atual.

Clique em “Rodar diagnóstico” para verificar a base atual do Supabase.

Ações que precisam de proteção real

O front-end pode esconder botões, mas o Supabase/RLS precisa bloquear ações indevidas de verdade.

  • profilesUsuário edita só o próprio perfil; Admin Master pode moderar.
  • teamsDono/capitão edita a própria equipe; Admin Master pode verificar/moderar.
  • team_membersEntrada em equipe só por convite aceito ou ação administrativa.
  • team_invitesCapitão/manager convida; jogador convidado aceita ou recusa.
  • organizer_permissionsApenas Admin Master/Admin SBW libera criação de Organizações de Torneios.
  • site_permissionsApenas Master Admin concede permissões altas.
