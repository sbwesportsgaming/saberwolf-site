# v1.5.8.2 — Slug único automático para torneios

Esta etapa evita erro de chave duplicada em `public.tournaments.slug` quando um organizador cria torneios com nomes iguais ou parecidos.

## Alteração principal

Arquivo alterado:

- `js/tournaments/tournaments-storage.js`

## Fluxo novo

Ao criar torneio no Supabase:

1. O sistema normaliza o nome do torneio para gerar o slug base.
2. Consulta o Supabase para saber se o slug já existe.
3. Se existir, tenta automaticamente:
   - `torneio`
   - `torneio-2`
   - `torneio-3`
   - etc.
4. Se ainda houver colisão por corrida de gravação, tenta uma correção final com sufixo temporal curto.

## Exemplo

Se já existir:

```txt
torneio-supabase
```

E o organizador criar outro torneio com o mesmo nome, o novo slug deve virar:

```txt
torneio-supabase-2
```

## Observação

Não precisa rodar SQL novo para esta etapa. A tabela `public.tournaments` precisa continuar tendo `slug text unique` ou constraint equivalente.
