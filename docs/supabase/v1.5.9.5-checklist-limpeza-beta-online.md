# v1.5.9.5 — Checklist e limpeza para Beta Online

Objetivo: preparar a versão atual para teste online fechado no domínio `sbwgg.com.br`, sem adicionar recurso competitivo novo.

## Alterações do patch

- Criada a página interna de diagnóstico: `/beta/beta-checklist.html`.
- Criado o script `js/beta/beta-checklist-page.js` para conferir configuração do front, Supabase, Auth, tabelas públicas e links principais.
- Criado o CSS `css/beta/beta-checklist.css`.
- Corrigido `robots.txt` para impedir indexação durante o beta fechado.
- Normalizada a pasta de imagens para `assets/images`, evitando erro de case sensitive em hospedagem Linux/GitHub Pages/Vercel/Netlify.
- Adicionados fallbacks temporários para avatars antigos de creators, evitando imagem quebrada no beta.
- Corrigidos links legados importantes em páginas de creators, comunidades, transferências, atletas e páginas antigas de torneios em subpastas.
- Adicionado `/beta/` à detecção de raiz dos helpers de Auth.
- Criado SQL de diagnóstico seguro: `docs/supabase-beta-online-v1.5.9.5-checks.sql`.

## Como testar localmente

1. Abrir `/beta/beta-checklist.html`.
2. Confirmar que o cliente Supabase inicializa.
3. Confirmar que as tabelas críticas respondem pela API REST.
4. Abrir os links principais indicados no checklist.
5. Testar login e reset de senha novamente.

## Configuração necessária no Supabase Auth

Em `Authentication > URL Configuration`:

```txt
Site URL:
https://sbwgg.com.br
```

Adicionar redirects:

```txt
https://sbwgg.com.br/auth/callback.html
https://sbwgg.com.br/auth/redefinir-senha.html
https://www.sbwgg.com.br/auth/callback.html
https://www.sbwgg.com.br/auth/redefinir-senha.html
```

Manter redirects locais para teste:

```txt
http://localhost:5500/**
http://127.0.0.1:5500/**
```

## SQL de diagnóstico

Rodar no Supabase SQL Editor:

```txt
docs/supabase-beta-online-v1.5.9.5-checks.sql
```

Esse SQL não apaga dados. Ele lista:

- tabelas esperadas e status de RLS;
- policies cadastradas;
- índices únicos de inscrição duplicada;
- duplicidades ativas;
- participantes fake/teste;
- torneios recentes;
- diferença entre `current_participants` e contagem real;
- criadores de torneios recentes.

## Limpeza antes do teste externo

Antes de chamar usuários externos, confirmar:

- nenhum participante fake/teste ativo;
- índices únicos de duplicidade restaurados;
- RLS ativa em tabelas sensíveis;
- usuário comum não cria torneio;
- organizador aprovado cria/gerencia apenas seu fluxo;
- Admin/Master SBW consegue supervisionar;
- reset de senha funciona no domínio online;
- `robots.txt` bloqueia indexação enquanto o beta for fechado.

## Fluxo de validação beta real

Teste com:

- 1 Admin SBW;
- 1 organizador real;
- 2 a 4 jogadores reais.

Fluxo:

```txt
criar conta
→ login
→ organizador cria torneio
→ jogadores se inscrevem
→ organizador confirma check-in
→ organizador gera estrutura
→ organizador lança resultados
→ público vê resultados
→ organizador finaliza torneio
→ perfis/histórico/rankings refletem o torneio
```

## Observação

Rankings por filtro, temporadas avançadas, aprovação de ranking global SBW, upload real de imagens e recursos extras continuam como melhorias posteriores. Para beta fechado, o foco é validar o ciclo principal com segurança.
