# v1.5.8.3 — Inscrições reais em torneios

Esta etapa fecha a base essencial para inscrição real de jogador logado em torneio salvo no Supabase.

## O que muda

- Mantém a inscrição vinculada ao usuário autenticado pelo Supabase Auth.
- Usa o perfil criado/encontrado por `SBWAuth.ensureCurrentUserProfile()`.
- Salva em `public.tournament_participants`.
- Evita inscrição duplicada ativa do mesmo usuário no mesmo torneio.
- Atualiza automaticamente `public.tournaments.current_participants` por trigger.
- Melhora mensagens de erro para tabela ausente, RLS e duplicidade.

Execute no Supabase SQL Editor:

## Teste recomendado

1. Garanta que `js/supabase/supabase-config.js` está com `enabled: true`.
2. Faça login em `/auth/login.html`.
3. Abra um torneio real vindo do Supabase em `/torneios/detalhe-torneio.html?id=slug-do-torneio`.
4. O botão deve aparecer como `Confirmar inscrição`.
5. Confirme a inscrição.
6. Verifique no Supabase:

```sql
select tournament_id, tournament_name, player_name, status, check_in_status, created_at
from public.tournament_participants
order by created_at desc;
```

7. Verifique a contagem:

```sql
select title, slug, current_participants
from public.tournaments
order by created_at desc;
```


## Patch 1 aplicado nesta revisão

- Corrigido o trigger `sbw_sync_tournament_participants_count()` para tratar `INSERT`, `UPDATE` e `DELETE` separadamente. Isso evita referência a `NEW` em operação de `DELETE`.
- A busca de participantes agora tenta localizar por `tournament_id` e por `tournament_slug`, ajudando em torneios acessados pelo slug público.
- A verificação de inscrição do usuário considera apenas inscrições ativas (`registered`/`waitlist`) e usa `limit(1)` para evitar erro caso exista histórico futuro de cancelamento/reinscrição.
- Adicionado índice único extra por `(tournament_slug, auth_user_id)` para reforçar o bloqueio de inscrição duplicada quando o slug público estiver preenchido.
