# v1.5.8.4 — Gestão de inscrições reais em torneios

## Objetivo

Esta etapa começa a transformar a inscrição real em painel de organização:

- listar torneios reais do Supabase no painel `criar-torneio.html`;
- abrir o gerenciamento de inscritos reais;
- carregar participantes da tabela `tournament_participants`;
- permitir check-in/desfazer check-in pelo organizador;
- permitir remoção lógica do inscrito com `status = removed`;
- manter `current_participants` sincronizado pelo trigger da v1.5.8.3;
- adicionar RLS para permitir update pelo criador/gestor do torneio.

## Arquivos alterados

- `js/tournaments/tournaments-storage.js`
- `js/tournaments/tournament-admin-page.js`
- `torneios/create-tournament/criar-torneio.html`
- `docs/supabase-tournament-participants-v1.5.8.4.sql`

## Como testar

1. Rodar no Supabase SQL Editor:

```sql
-- conteúdo completo de docs/supabase-tournament-participants-v1.5.8.4.sql
```

2. Entrar no site com o mesmo usuário que criou o torneio.
3. Abrir:

```txt
/torneios/create-tournament/criar-torneio.html
```

4. Na lista de torneios, clicar em **Gerenciar inscritos** no torneio real do Supabase.
5. Confirmar que os inscritos reais aparecem.
6. Clicar em **Confirmar check-in**.
7. Conferir no Supabase:

```sql
select player_name, status, check_in_status, updated_at
from public.tournament_participants
order by updated_at desc;
```

8. Clicar em **Remover** em um inscrito de teste.
9. Conferir que o participante muda para `status = removed` e que `current_participants` é recalculado.

## Observações

- A geração de estrutura com inscritos reais do Supabase ficou desativada nesta etapa para evitar misturar bracket local com dados reais antes de salvar estrutura no banco.
- A remoção não apaga a linha; usa status `removed`, preservando histórico/auditoria.
