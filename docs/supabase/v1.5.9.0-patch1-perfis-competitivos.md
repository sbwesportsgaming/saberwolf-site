# v1.5.9.0 patch 1 — Perfis competitivos visíveis

Este patch corrige a primeira validação do histórico competitivo pós-torneio.

## Correções

- `perfis/perfis.html` voltou a carregar os scripts necessários da listagem pública:
  - `profiles-models.js`
  - `profiles-storage.js`
  - `profiles-list-page.js`
- A listagem pública de perfis agora também pode exibir perfis competitivos virtuais derivados de torneios finalizados.
- Quando um jogador aparece em `settings.finalResults.placements`, mas ainda não possui perfil público cadastrado no Supabase, a plataforma cria um perfil visual temporário para teste/visualização.
- `perfil.html?id=...` agora consegue abrir um perfil virtual de jogador presente no resultado final oficial.
- O snapshot competitivo ficou mais tolerante: além de buscar na tabela `tournament_participants`, também cruza diretamente com os IDs/slugs/nomes salvos em `finalResults.placements`.
- Novos fechamentos de torneio passam a preservar `profileId`, `playerSlug` e `authUserId` nos placements finais quando esses dados existirem na estrutura.

## Observação importante

A aba `Meu Perfil > Medalhas` só mostra medalhas do usuário logado quando o perfil logado realmente aparece no resultado final do torneio.

Se o torneio foi testado com participantes fake e depois esses fakes foram limpos, é normal o usuário real não receber medalha. Nesse caso, a visualização pode ser testada pela listagem pública de perfis competitivos gerados a partir de `finalResults`.

## Como testar

1. Abrir `/perfis/perfis.html`.
2. Confirmar que aparecem perfis reais do Supabase e/ou perfis competitivos derivados do torneio finalizado.
3. Clicar em um perfil vindo do resultado final.
4. Confirmar que aparecem medalhas e histórico competitivo.
5. Para testar `Meu Perfil`, finalize futuramente um torneio em que o usuário logado seja um dos participantes reais.

## SQL

Não precisa rodar SQL novo.
