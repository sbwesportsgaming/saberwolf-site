# v1.5.9.4 — Preparação Beta Real: recuperação de senha

## Objetivo

Preparar a plataforma -SBW- para beta online real com fluxo básico de recuperação/redefinição de senha usando Supabase Auth.

## Arquivos alterados/adicionados

- `auth/login.html`
- `auth/esqueci-senha.html`
- `auth/redefinir-senha.html`
- `js/auth/auth-service.js`
- `css/auth.css`

## Fluxo implementado

1. Usuário clica em `Esqueci minha senha` no login.
2. Usuário informa e-mail em `auth/esqueci-senha.html`.
3. Front chama `supabase.auth.resetPasswordForEmail(email, { redirectTo })`.
4. Supabase envia e-mail de recuperação.
5. Link redireciona para `auth/redefinir-senha.html`.
6. Usuário informa nova senha.
7. Front chama `supabase.auth.updateUser({ password })`.
8. Usuário é deslogado e volta para o login.

## Configuração necessária no Supabase antes do beta online

Em `Authentication > URL Configuration`:

- Site URL: `https://sbwgg.com.br`
- Redirect URLs:
  - `https://sbwgg.com.br/auth/callback.html`
  - `https://sbwgg.com.br/auth/redefinir-senha.html`

Enquanto estiver testando localmente, manter também:

- `http://127.0.0.1:5500/**`
- `http://localhost:5500/**`

Se usar `www`, adicionar também:

- `https://www.sbwgg.com.br/auth/callback.html`
- `https://www.sbwgg.com.br/auth/redefinir-senha.html`

## Observação de segurança

Supabase Auth cuida da autenticação, sessão, tokens e fluxo de reset. A plataforma -SBW- continua responsável por:

- RLS das tabelas públicas/privadas.
- Não expor chave `service_role` no front-end.
- Configurar domínios e redirects corretos.
- Mensagens, UX e regras de permissão da aplicação.
- Revisar quem pode editar torneios, equipes, perfis e inscrições.
