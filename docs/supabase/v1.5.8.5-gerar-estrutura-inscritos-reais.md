# v1.5.8.5 — Gerar estrutura com inscritos reais

Esta etapa libera a geração de estrutura para torneios reais vindos do Supabase.

## O que muda

- O botão **Gerar estrutura do torneio** fica disponível também para torneios Supabase.
- Antes de gerar, o painel recarrega os inscritos reais da tabela `tournament_participants`.
- Só entram na estrutura participantes com `status = registered`.
- Se o check-in do torneio estiver como `required`, só entram participantes com `check_in_status = checked_in`.
- A estrutura gerada é salva no `settings` JSONB do torneio real:
  - `settings.structure`
  - `settings.matches`
  - `settings.standings`
  - `settings.structureGeneratedAt`
  - `settings.structureGeneratedFrom = real-registrations`
- O status do torneio muda para `structure-generated` quando aplicável.

## Arquivos alterados

- `js/tournaments/tournament-admin-page.js`
- `js/tournaments/tournaments-storage.js`

## Teste recomendado

1. Abrir `/torneios/create-tournament/criar-torneio.html`.
2. Clicar em **Gerenciar inscritos** no torneio Supabase.
3. Confirmar check-in em pelo menos 2 participantes reais.
4. Clicar em **Gerar estrutura do torneio**.
5. Confirmar a geração.
6. Verificar se a estrutura aparece no painel.
7. Conferir no Supabase:

```sql
select
  title,
  slug,
  status,
  settings->'structure' as structure,
  settings->'matches' as matches,
  settings->'standings' as standings,
  settings->>'structureGeneratedFrom' as generated_from,
  settings->>'structureGeneratedAt' as generated_at
from public.tournaments
order by updated_at desc;
```

## Observação

Nesta etapa a estrutura é persistida dentro do JSONB `settings`. Tabelas separadas para `tournament_matches`, `tournament_sets` e auditoria detalhada ficam para uma etapa futura.
